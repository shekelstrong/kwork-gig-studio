# Kwork Gig Studio

Единый пайплайн создания кворка на Kwork. От ниши до публикации за 15 минут.

## Когда использовать

- Юзер говорит «нужен кворк», «создай кворк», «опубликуй на Kwork»
- Юзер присылает описание услуги или нишу
- Юзер хочет серию кворков в одной нише
- Нужно переоформить старый кворк под новые правила модерации

## НЕ использовать

- Юзер просто хочет обсудить ценообразование (это про pricing-strategy, не про создание)
- Ниша явно вне Kwork (B2B enterprise с бюджетом 500к+)

## Что внутри

5 MCP tools, объединённых в один pipeline:

```
ниша → create_gig_draft → generate_cover → validate_cover → check_tos → publish_gig
       ↓                   ↓                ↓                ↓            ↓
   текст+FAQ         PNG 1024x1024    bool+issues      violations    URL кворка
```

## Алгоритм

### 1. Спроси входные данные (если их нет)

```yaml
ниша: "AI-аудит заявок для интернет-магазинов"
цена: 5000
срок: 1 день
аудитория: "владельцы магазинов с 50+ заявок/день"
```

Если юзер не знает цену — предложи 4000-8000₽ для нового кворка (минимум 400₽ на Kwork, шаг 100₽).

### 2. Сгенерируй черновик (`create_gig_draft`)

- **Название** ≤ 70 символов, с конкретной выгодой
- **Описание** 100-1200 символов, структура: что делаю → для кого → срок → результат
- **"От покупателя нужно"** ≤ 500 символов
- **FAQ** 3-5 вопросов с короткими ответами

### 3. Сгенерируй обложку (`generate_cover`)

Стиль dark-saas по умолчанию:
- Фон #0a0a0a → #161616
- Акценты mint #7ee787 + sky #79c0ff
- 50/50 layout: заголовок+метрики слева, иллюстрация справа
- 3 буллита с зелёными ✓
- Без лого Kwork, без брендов, без цен, без триггер-слов

### 4. Проверь обложку (`validate_cover`)

⚠️ **КРИТИЧНО** — это решает пройдёт ли модерация.

Vision-чекер проверяет:
- Бренды (VK, Avito, hh.ru, Ozon, WB, ...) — бан
- Цены на обложке — бан
- Триггер-слова (рилс/Reels/сторис/паблик/SMM) — бан
- < 70% кириллицы — бан

Если `ok=false` — перегенерируй обложку с другим промптом.

### 5. Проверь текст (`check_tos`)

Те же правила, но для текста. Плюс:
- Название > 70 — обрежь
- Описание > 1200 — сократи
- "От покупателя нужно" > 500 — сократи

### 6. Опубликуй (опционально, `publish_gig`)

⚠️ Только если юзер явно просит. По умолчанию — нет. Лучше показать черновик, юзер сам нажмёт "Опубликовать".

## Pitfalls

| Ошибка | Последствие | Как избежать |
|---|---|---|
| Пропустить `validate_cover` | Кворк завернут модерацией | ВСЕГДА запускай |
| Использовать "рилс" в нише SMM | Бан (триггер-слово) | "короткое видео" вместо "рилс" |
| Упомянуть VK в нише "Парсинг" | Бан (бренд) | "соцсеть" или конкретное нишевое название |
| Написать цену на обложке | Бан | Только в полях цены кворка, не на картинке |
| Загрузить обложку < 660x440 | Не пройдёт | Генерируй минимум 1024x1024 |
| Использовать TODO в тексте | Юзер в панике | Только готовые формулировки |

## Пример полного pipeline

```python
# Вход
niche = "Telegram-бот с ИИ для квалификации лидов"
price = 8000
deadline = 5  # дней

# 1. Черновик
draft = create_gig_draft(niche=niche, price=price, deadline_days=deadline)
# → {"title": "Сделаю Telegram-бота с ИИ...", "description": "...", "faq": [...]}

# 2. Обложка
cover = generate_cover(
    title=draft["title"],
    metrics=["5 дней", "8 000 ₽", "Готов к запуску"],
    bullets=["LLM-агент", "aiogram 3", "Docker"]
)
# → "/tmp/covers/kwork-leads-bot-2026-07-03.png"

# 3. Vision-чек
check = validate_cover(cover)
if not check["ok"]:
    cover = regenerate_cover_with_new_prompt(...)
    check = validate_cover(cover)

# 4. ToS-чек
tos = check_tos(draft["description"], check_type="description")
if tos:
    draft = fix_tos_violations(draft, tos)

# 5. Показать юзеру
return {
    "title": draft["title"],
    "description": draft["description"],
    "price": price,
    "cover": cover,
    "ready_to_publish": True
}
```
